1.1 Purpose

((The purpose of this document is to describe the web application Obesity Prevalence Visualizer, giving details about the Interface, the system features, etc.))The scope of the product is to provide information to the user on the situation of obesity cases all over the world. On the site, the statistics are displayed according to certain criteria chosen by the user.

1.4 Product Scope

OPreV (Obesity Prevalence Visualizer) is a web application that helps the people to see the latest statistics available about obesity rate in certain countries, the data being selected after some criteria that the user is choosing. Also, the application has a special page, in order to help people who need motivation by announcing them about events that may help them.

3. External Interface Requirements

3.1 User Interfaces

On each page, besides the login page, has a menu, in order to help the user to navigate between the pages. The menu buttons are “Statistici”, “Evenimente”, “Despre noi” and “Contul meu”.

The main page is the statistics page, where the user has at his disposal a wide variety of filters, in order to see the rate of obesity in the world (examples of filters: age, sex, country, etc..). The user can download this data in various formats such as : svg, png, csv and can also change the display format.

The event page contains information about eveniments taking place online or on site, in order to help a person who is overweight, obese or a person who would like to have a healthy life style to get motivated.

The about us page has some basic information about our purpose stages of obesity and some recommendations.

The login page is the page where the administrator can sign in; he has to introduce the email and the password.

The register page is a page where an administrator cand sign up, he has to wait for the principal administrator to accept his request.

Another page is the one where the administrator can see the information about his account and also has buttons for editing data on the website and one for log out.

There exists a page where the administrator can change the events info, where he can add an event or remove an event or he can remove types of filter in case that there exists inconsistent data.

4. System Features

4.1 Statistics Management

The site is equipped with a data filtering section, so that each person who accesses can view statistics based on the filters they have chosen. Moreover, the data can be viewed in several ways such as PDF , PNG, CSV so that it is easier for the user to manage the received data.

4.2 Editing data by the administrator

The administration part of the site comes with a feature that gives the admin the ability to edit the filters in case a problem happens, to publish new events or delete the already finished ones.
