1. Introduction

1.1 Purpose

The scope of the product is to provide information to the user on the situation of obesity cases all over the world. On the site, the statistics are displayed according to certain criteria chosen by the user.

The main objective of this document is to describe the web application Obesity Prevalence Visualizer, giving details about product scope, functions, the interface, the system features, etc.

1.2 Document Convention

The documentation for this project respects the IEE System Requirements Specification Template.

1.3 Intended Audience and Reading Suggestions

This project is useful for any person who is interested in seeing statistics about obesity cases in their country or any other country and documentation for an user who would like to understand better the application and its purposes.

Also, this documentation is useful for programmers who are interested in improving this application.

For an user, the most important section of this documentation is the introduction part and the user interface part. For a programmer, it is important to read the whole document in order to understand better the scope and the organization of implementation

1.4 Product Scope

OPreV (Obesity Prevalence Visualizer) is a web application that helps the people to see the latest statistics available about obesity rate in certain countries, the data being selected after some criteria that the user is choosing, for example country, sex, age, etc. Here, the user can view the data in different modes and can download it, in certain formats : csv, png and svg.

Also, the application has a special page, in order to help people who need motivation by announcing them about events that may help them.

2. Overall Description

2.1 Product Perspective

The OPreV is an application for everyone who is interested in overweight statistics and events related to that field.

The OPreV is following the example of eurostat and who, websites that displays statistics for a lot of topics, and reproduces the topic about overweight cases over the world.

2.2 Product Functions

The application will let the user to choose some filters(for example: age, country, sex) in order to see the distribution of the obesity cases and view them in different formats, then there exists the possibility of download them. Also, the application will inform the user about new events all over the world(online or on site) in order to motivate him/her to have a healthy life.

Also, the user can see information about the main scope of the application. For an administrator, there will be two functions in plus; he will be able to add or to erase events or filters and to see information about his own account.

2.3 User Classes and Characteristics

There are two types of users who use the application :

  • typical user, a person who is interested in knowing more about the rate of obesity or events related to this problem;
  • administrator, that has permission to edit data in order to keep the information accurate and to be sure that there are no mistakes, for example, he can add events, remove event, add types of filters or delete types of filter;

2.4 Operating Environment

  • database: phpmyadmin, sql
  • operating system: Windows
  • platform: Php

2.5 Design and Implementation Constraints

The application is written in php, plus its useful tools for the backend part and the frontend part is in css and html.

3. External Interface Requirements

3.1 User Interfaces

On each page, besides the login page, has a menu, in order to help the user to navigate between the pages. The menu buttons are “Statistici”, “Evenimente”, “Despre noi” and “Contul meu”.

The main page is the statistics page, where the user has at his disposal a wide variety of filters, in order to see the rate of obesity in the world (examples of filters: age, sex, country, etc..). The user can download this data in various formats such as : svg, png, csv and can also change the display format.

The event page contains information about eveniments taking place online or on site, in order to help a person who is overweight, obese or a person who would like to have a healthy life style to get motivated.

The about us page has some basic information about our purpose stages of obesity and some recommendations.

The login page is the page where the administrator can sign in; he has to introduce the email and the password.

The register page is a page where an administrator cand sign up, he has to wait for the principal administrator to accept his request.

Another page is the one where the administrator can see the information about his account and also has buttons for editing data on the website and one for log out.

There exists a page where the administrator can change the events info, where he can add an event or remove an event or he can remove types of filter in case that there exists inconsistent data.

4. System Features

4.1 Statistics Management

The site is equipped with a data filtering section, so that each person who accesses can download statistics based on the filters they have chosen. Moreover, the data can be viewed in several ways such as PDF , PNG, CSV so that it is easier for the user to manage the received data.

4.2 Editing data by the administrator

The administration part of the site comes with a feature that gives the admin the ability to edit the filters in case a problem happens, to publish new events or delete the already finished ones.
